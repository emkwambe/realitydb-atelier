## Manual migration — run in Supabase SQL Editor

```sql
CREATE TABLE public.waitlist (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  email text UNIQUE NOT NULL,
  source text,
  referrer text,
  created_at timestamptz NOT NULL DEFAULT now()
);

ALTER TABLE public.waitlist ENABLE ROW LEVEL SECURITY;

-- Anyone can insert their own email
CREATE POLICY "waitlist_insert_anon" ON public.waitlist
FOR INSERT
WITH CHECK (true);

-- Only admins can read the list
CREATE POLICY "waitlist_select_admin" ON public.waitlist
FOR SELECT
USING (
  EXISTS (
    SELECT 1 FROM public.profiles
    WHERE id = auth.uid() AND role = 'admin'
  )
);
```

## What shipped

  inserts via service-role client, treats duplicate-email as success
  so the user never sees an error. Per-IP rate limit (3/hour) via an
  in-process Map.

  link alongside the existing "Admin" link.
